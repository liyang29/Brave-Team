## 模拟驱动的难度校准（用法 A / B）

**来源**：自动战斗是"可被纯模拟的函数"——能在开打前跑 N 次蒙特卡洛预演、算出当前队伍对某组敌人的胜率。`test_balance.gd` 已具备这个能力。

**评价：好，但必须分清用法。** 核心能力(可模拟→可测胜率)是本作设计的独特红利；危险的是"拿它锁死胜率"。

### ✅ 用法 A：离线"调参尺子"（推荐，已落地雏形）
对一个"基准 build"逐关跑模拟、报胜率 + 阵亡分布，**人工据此调固定难度表**。难度仍是作者定的（固定），只是用数据校准，不靠拍脑袋。**build 好仍碾、build 烂仍死 → 火花保住。**
- 实现：`test_balance.gd::test_per_node_health_report`（逐关满血胜率 + 中庸 build 整局阵亡分布）。
- **首次发现**：当前每关满血单独打全是 20/20（单场都不难），难度 100% 在"消耗战(跨关掉血)"；中庸 build 失败时**全死在魔王**——魔王是唯一的难度墙，其余是消耗热身。

### 🟡 用法 B：宽带"安全网" DDA（以后可选）
开打前预演，**只排除极端局**：胜率 <~15%(几乎必死) 或 >~98%(毫无挑战) 才从备选怪池换一组；**中间留宽带(如 30%~95%)放行**。DDA 只设护栏、不当调速器——神 build 照样碾、烂 build 照样挣扎，落差还在，只是不会出现纯劝退死局。

### ❌ 反模式：硬锁胜率（隐藏橡皮筋）
不管玩家搭得多好都缩放敌人维持固定胜率（如恒 80%）。**会抹掉"build 决定成败"**：神 build 不爽、烂 build 没代价 → 浇灭核心火花；且 roguelike 玩家能闻出橡皮筋。**不要做。**（违背 METHODOLOGY「选错有代价」。）

**业界参照**：动态难度老技术(生化危机4/马里奥赛车)；按状态调遭遇有先例(求生之路 AI Director，但用启发式非模拟)；模拟做平衡是卡牌/自走棋行业标准(多为离线)；roguelike 传统是**固定难度**(杀戮尖塔/FTL/NetHack)，哈迪斯"上帝模式"是**可选知情**的辅助 DDA。本作的"实时蒙特卡洛预演校准"组合少见，是"战斗可模拟"带来的红利。

**难度梯度设想**（待定，注意"单场 vs 整局"复利：单场 80%×4 场 ≈ 整局 41%）：极难≈60% / 一般≈80–90% 通关率。

**下一步**：先把用法 A 当常规调参流程；B 留作防劝退安全网；A/B 之外别碰硬橡皮筋。

## 数值架构蓝图（1→100 关的扩展 + 心流）

**要解决的核心问题**：内容会膨胀（无限装备/怪物）、关卡会拉长（不止 7 关），需要在数值上让难度可扩展、又让玩家保持心流。**从 Demo 走向大游戏绕不开的地基。**

### 目标先摆正：要的是"比值锁带"，不是"线性数值"
决定心流的不是绝对数值，是 **`敌人战力/我方战力` 的比值**（心流通道：挑战≈能力）。双方一起涨，只要比值待在目标带（普通关→高胜率、Boss→吃紧），难度就恒定；绝对数字是"皮肤"。纯线性手感差、纯指数数字爆——**盯比值，别盯线**。

### 武器 1：战力分 / 预算（应对内容膨胀）
给**万物一个单一数字**：装备 power=属性加权和；英雄+背包 power=f(base+装备+协同)；怪物 power≈EHP×DPS；遭遇=怪物 power 之和。
→ 加新内容 = 标个 power 落在曲线上；组遭遇 = 按"预算"凑怪。**不再手调每关每怪。**
血脉：D&D 的 CR / MMO 装等 iLvl / 暗黑 Gear Score。

### 武器 2：数学
- **逻辑斯蒂/Elo 把战力差换算胜率**：`P(胜)≈1/(1+10^((敌power−我power)/S))`。反解"想要的胜率"→"该给多少敌人 power"。
- **缩放**：`敌人预算(d)=目标比值 × 预期我方战力(d)`；普通关比值低、精英/Boss 高。
- **边际递减/软上限**（稳定器）：属性叠加收益递减，**收窄玩家战力方差**，否则 roguelike 运气方差让任何曲线失控。

### roguelike 的难点 + 解法
难点：玩家战力是**随机变量**（看抽到/怎么搭），"第 d 关多强"方差巨大——这是数值难的根源。
- 路 A：固定曲线 + 接受方差（杀戮尖塔/FTL，靠"强 build 走得远"自然匹配）。
- 路 B：软上限+保底掉落 **收窄方差**，固定曲线就够。
- 路 C（本作独有）：**战斗可模拟 → 实时测真实战力 → 校准下一关预算**（用法 A/B；务必宽带别锁死）。
- **推荐组合 = B + A 打底 + C 当尺子/宽带安全网。**

### 心流是锯齿，不是平线
普通关让你变强(~90%)→ 精英/Boss 设墙(~60%)→ 打过→ 再循环。别全程压一个数。体检表已显示本作天然是这形状（魔王是唯一墙）。

### 100 关靠"生成"不靠"手摆"
关卡程序生成（吃 群系配置 + 每深度 power 预算）；怪物池按 tier 标 power 按预算凑；装备按 power 曲线 + 深度门控掉落。→ **平衡 = 调几条曲线参数，不是调 100 关。**

### 诚实边界
roguelike 无法保证每个玩家心流不断（方差是 genre 灵魂）；目标是把带控在合理范围、不是完美直线；**别硬锁胜率**（杀 build-matters）。

### 已落地：PowerScore 用法
`const Power = preload("res://scripts/systems/PowerScore.gd")`

| 函数 | 作用 | 尺度 |
|------|------|------|
| `unit_power(hp,atk,def,magic,spd, crit_chance=0, crit_dmg=0)` | 通用单位战斗评级 = 有效血量×输出×速度 | 大（百~千） |
| `enemy_power(EnemyData)` | 怪物战力（套 unit_power） | 同上 |
| `group_power([EnemyData...])` | 一组怪总战力 = 遭遇"预算" | 同上 |
| `hero_power({base, grid})` | 英雄含背包战力（裸base+背包compute，暴击折进输出） | **同 enemy_power** |
| `team_power([名册条目...])` | 队伍总战力 = 各人之和 | 同上 |
| `item_power(item_id)` | 单件装备属性加权分（防>血、暴击按百分点、技能书基础分） | 小（几~几十），**仅用于排序/定价/掉落，别和单位分比** |

**关键**：`hero_power`/`team_power` 与 `enemy_power`/`group_power` **同尺度**，可直接求难度比值：
```gdscript
var ratio = Power.group_power(node.enemies) / Power.team_power(RunManager.roster)
# ratio 越大越难；配合逻辑斯蒂可换算成预期胜率
```
**用途**：① 遭遇预算生成(按 group_power 凑怪到目标比值) ② 掉落/商店按 item_power 分级定价 ③ 喂 harness 校准曲线。
**注意**：所有权重(`W_DEF` 等)与系数(`DEF_TO_EHP` 等)都是可调旋钮，先用直觉值、后用 harness 校准；`item_power` 与单位分量纲不同，不可混比。

**一句话**：万物给 power 分(应对膨胀) → 逻辑斯蒂锁胜率带(应对心流) → 软上限收方差 + 模拟当尺子(应对随机) → 程序生成关卡(应对 100 关)。
