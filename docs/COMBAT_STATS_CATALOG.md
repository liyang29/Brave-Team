## 8. 触发器 / proc（这个品类真正的深度引擎）

> 重点：Megaloot / Backpack Battles 的深度很大程度在 **proc 的相互咬合**（"暴击时叠毒 → 击杀回蓝 → 再放技能"这种连锁），比堆裸属性更出彩。我们已有 on-hit / on-kill / on-taken 钩子，最该深挖。

| 触发时机 | 例子 | 现状 |
|----------|------|------|
| 命中时 on-hit | 命中叠毒 / 回血 | ◐（钩子有，效果待扩） |
| 暴击时 on-crit | 暴击叠 buff | ➕ |
| 击杀时 on-kill | 击杀回血 / 加攻 | ✓（lifesteal） |
| 受击时 on-hit-taken | 反伤 / 受击加盾 | ✓（thorns） |
| 回合开始 / 每 N 回合 | 周期充能 / 回血 | ➕ |
| 残血时 on-low-hp | 触发保命 | ➕ |
| 开场 on-battle-start | 首回合增益 | ➕ |
| 放技能时 on-cast | 连锁触发 | ➕ |

## 加属性时的纪律

每加一个战斗数据，先问一句：**"它能造出一种新 build 或新克制吗？"**
- 能 → 加；只是"让数字更大" → 不加。
- 先上少数几个验证好玩，再像 Megaloot 那样慢慢堆到几十个。
- **属性多不等于好玩，属性之间能咬合（proc 连锁）才好玩。**

### 建议的第一波（低成本 × 高 build 区分度）
1. ~~暴击率 + 暴击伤害~~ ✅ **已实现**（首个住进 `extra_stats` 字典的副属性）
2. 法术抗性（开启 物理↔魔法 克制轴）
3. 吸血%（泛化现有 on-kill 触发为属性）
4. 破甲（把技能 ignore_def 升为属性，克重甲）

> **地基已铺好**：`BattleCombatant.extra_stats` 字典 + `BackpackModel.EXTRA_KEYS`。加上面 2~4 这类 A 档纯数字副属性 ≈ 扩 `EXTRA_KEYS` + 物品声明 key + 公式读 `get_stat()`，不动结构。
