# 思想方法（设计判断的指南针）

> 这是做这个游戏时反复用到的判断框架，从"上一个游戏为什么不好玩"的反思里提炼出来。做新功能、做取舍时回来对照。

## 1. 无聊的根因：执行已知流程，而非做决策

上一个游戏（公会经营）不好玩，根因不是美术，而是：**玩家在"执行一套已知的最优流程"，而不是"做有意义的抉择"，且做错也没代价。**

人脑觉得好玩 = 面对**不知道答案、答错有后果**的局面被迫思考。所以：

- **新鲜感 ≠ 深度**。开罗游戏式的"不停解锁新内容"是新鲜感（前两小时上头），一旦看完内容就空了。**真深度 = 知道全部规则后还耐玩**（因为最优解随局面一直变）。
- 杀死深度的三件事：① 存在唯一主导策略（最优解固定）② 没有真风险（数值只升不降）③ 局面不随上下文变（解一次终身复用）。

## 2. 真抉择三要素（加任何东西先过这关）

一个设计要"有意思"，得让玩家做**有意义的抉择**：

1. **有取舍**：选项之间没有全方位最优（带技能书=少带护甲；摆这格安全但挨穿刺）。
2. **随上下文变**：每场遭遇/每局的最优解不同，逼你重新想。
3. **有真代价**：选错会失去你在乎的东西（永久死亡 + 丢整个背包）。

> **加一个属性/机制前，先问：它能造出一种新 build 或新克制吗？** 能 → 加；只是"让数字更大" → 不加。

## 3. 只做深一个系统

深度引擎只能有一个主心骨。本作 = **背包构筑**。

- 战斗刻意做**轻**（自动结算），深度全压背包——Megaloot 验证可行。
- 站位、技能书、副属性都**并入或服务于背包**这一个系统，不另起一个深空间谜题（曾否决"深背包 + 深站位网格"同时做）。
- 警惕第二个深系统来抢玩家的脑子（如把 AI 策略做成 FT12 Gambit 那种编程谜题）。

## 4. 回合制 + 自动结算（不做实时）

- 回合制**可读、可测、好平衡**；实时会让"谜面"变模糊、且要重写引擎。
- "实时观感"留给后期**动画播放回合结果**，逻辑层始终回合制。
- 想要"战斗中自己出招"是另一种游戏（Megaloot 式手动）；本作选**自动**，把决策放在战前构筑。

## 5. 脏实验：先验证好不好玩，再建系统

- **白盒优先**：用色块/文字先把"核心循环跑通"，真实玩几把判断好不好玩。便宜地发现"不好玩"，是白盒的全部意义。
- **脏实验**：独立场景、只验证**一个**核心乐趣假设、可整体回退。先做最小切片感受"火花"，有火花再正式铺。
- 例：背包脏实验只问一句——"搭出一套组合赢下来，有没有'我真聪明'的火花？"

## 6. 自动战斗的公平性：可预测 > 可控制

战斗自动时，玩家的成就感来自"我搭的 build 起作用了"。前提是**自动结算可预测**（玩家能推理"这样搭→会这样打"）。

- 先让默认 AI**够聪明、够可读**（治本），再给玩家**少量策略旋钮**（安全阀）。
- 目标是让玩家"怪自己策略没设好"，而不是"怪 AI 瞎打"。

## 7. 工程纪律

- **可扩展地基优先**：如 `extra_stats` 字典——铺一次，以后加副属性≈改数据。
- **向后兼容**：新机制默认不改变旧行为（如站位 `positioning_mode` 默认 reach、副属性默认空）。
- **每个改动配测试**，全量绿再合并。
- 继承深度 ≤ 3 层，超过改组合；数据驱动，数值不硬编码进逻辑。
