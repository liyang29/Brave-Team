# Build 设计（构筑深度从哪来）

> 本作的"深度引擎"= 背包构筑。本文件讲 build 由什么构成、深度怎么加、边界在哪。
> 配套：`ROGUELIKE_DESIGN.md`(方向)、`METHODOLOGY.md`(判断框架)、`COMBAT_MANUAL.md`(战斗规则)。

## 0. 核心洞察：背包把所有东西焊成"一个 build 空间"

**别把 build 拆成传统 RPG 的"技能树 + 装备栏"两套系统。** 在本作里，技能/装备/属性/协同**共用同一个背包格子**——它们在有限格子里**抢地盘**：

- 带斩击书 = 少一格放护甲（技能 vs 生存）。
- 凑开刃(剑+磨刀石相邻) = 占两个相邻格，挤掉别的。
- 暴击宝石占一格但不给攻防血。

所以一个英雄的 build = **「6 格里如何分配 属性/技能/协同/站位」**。技能和装备不是两层，是**同一张牌桌抢格子**。这是 Megaloot/Backpack Hero 的精髓，也守住"只做深一个系统"——深度全压背包，不另起技能树这个第二深空间。

## 1. build 的几条轴

| 轴 | 现状 | 定位 |
|----|------|------|
| 属性分配（装备 atk/def/hp/magic/mp/crit） | 有 | 通用 |
| 技能（技能书：占格 + 回合 CD + 认职业） | 有 | 通用 |
| **空间协同（相邻 tag 触发）** | 4 种配方 | ⭐ 招牌（Backpack Hero 式空间谜题）|
| 站位（前后排 + 近战/远程触及） | 有 | 半身份 |
| **小队跨英雄协同** | **无** | ⭐⭐ 差异化（vs 单角色背包大作）|

## 2. 三个深度方向（已定：三个都做）

### ⭐⭐ A. 小队跨英雄协同（差异化护城河，目前完全没有）
现在 3~5 个背包**各搭各的、互不影响** → 其实只是"3 个独立单人背包"，没体现"小队"。让**一个英雄的背包/站位影响别的英雄**，build 才从"3 个谜题"升级成"一个小队级谜题"：
- 全队增益物品："军旗"→ 全队 +攻；"号角"→ 全队 +速。
- 站位关联："圣坛"→ 给相邻站位的友军回血；前排"嘲讽"吸火力保后排。
- 角色互补：辅助型英雄的价值=放大别人，而非自己输出。
- **实现思路**：build_party 注入时，除"自身背包加成"外，再算一层"全队/相邻效果"（新 tag/字段，如 `team_atk`、`aura_heal`）。谁辅助谁、谁保谁 = 小队 build 的核心抉择。

### ⭐ B. 深化空间协同（招牌做厚）
4 种相邻配方太少。加：
- 更多 tag 配方：破甲、吸血、连击、过载…→ 更多空间谜题。
- 异形物品：占 2 格 / L 形 → 摆位像俄罗斯方块，空间取舍更硬。
- （**负空间/诅咒物品**：本轮决定**先不做**，保持"物品都是正收益"；以后想加真取舍时再上。）

### C. 克制关键词（让最优解随局面变）
现在克制太少，最优 build 几乎固定。加"关键词 × 敌人特性"的对位，让**每场遭遇的最优搭配不同**（真抉择第②要素）：
- 破甲/无视防御 ↔ 高防敌人；吸血 ↔ 持久战；净化 ↔ 中毒怪（已有 purify）；远程触及 ↔ 后排怪（已有）。
- 敌人也带特性（高防/剧毒/后排狙击），逼玩家**带对的答案**。

## 3. 真抉择三要素（build 才"有意思"）
1. **取舍**：格子有限（已有）+ 站位暴露 + 技能 vs 护甲。
2. **随上下文变**：敌人组成逼你换 build（靠 §2C 克制关键词）。
3. **真代价**：roguelike 永久死 + 丢背包（方向已定）。

## 4. 设计边界 / 原则
- **只做深背包一个系统**：所有深度通过背包(格子/相邻/站位/小队)表达，不加平行技能树。
- **暂不做负空间/诅咒**（本轮决定）：先把协同/克制/小队做厚。
- **不显式命名流派**（本轮决定）：靠数值 + 协同提示 + tooltip，让玩家**自己悟出**"我在走暴击/重装/法核"，不弹"暴击流"标签——保留"我真聪明"的发现感。
- **向后兼容**：新机制默认不改旧行为（新 tag/字段默认空）。
- **数据驱动**：协同配方、克制关键词、小队效果都进数据表，加 build 料 = 加数据行。

## 5. 流派范例（内部设计靶子，不在 UI 显示）
用来检验"是否有多种可行 build、是否互有取舍"——不是给玩家看的标签：
- **暴击战士**：blade + crit + 开刃，玻璃偏激进。
- **重装墙**：guard+armor 重装，高防高血，前排护后排。
- **法核**：arcane 共鸣 + 火球 + 法力护符，后排核弹，**需前排保护**(站位耦合)。
- **斩杀盗贼**：前排盗贼 + crit + finisher，收割残血。
- **中毒流**：DoT 叠加 + 拖延，克持久但怕净化/速攻。
- **小队增益核**(待 §2A)：一个辅助英雄放大全队 → 围绕它组队。
> 目标：这些都**可行且互有克制**，没有唯一主导 → 才有深度（METHODOLOGY §2）。

## 6. 落地优先级建议
1. **克制关键词(C)** 最便宜：多是给技能/物品加字段 + 给怪加特性，复用现有效果框架。先做能立刻让"build 随遭遇变"。
2. **小队跨英雄协同(A)** 价值最高（差异化）：需在 build_party 注入加一层"全队/相邻效果"，是新机制，值得专门设计。
3. **深化空间协同(B)** 持续加料：随内容慢慢扩 tag/配方；异形物品是较大的 UI/逻辑改动，靠后。

> 关联：build 强弱可用 `PowerScore`(IDEAS 蓝图)量化；新 build 料配 GUT + balance harness 验证"多 build 可行、无唯一解"。

## 7. 小队协同详设（四轴拆解 + 落地）

### 思路：小队 = "空间 × 协同"的队级谜题
现在 3 个背包各搭各的（=3 个单人背包）。小队的灵魂是两件事咬合：
1. **站位决定暴露与触及**（已有：前排挡刀、近战啃前线）。
2. **build 外溢**（新：某人带的东西加成别人，且**谁站哪决定加成给谁**）。
→ 决策从"每人怎么搭"升级成"**谁站哪、谁带队级物品、谁辅助谁、谁保谁**"。

### 最小颗粒度：任何小队机制 = 四个正交轴的组合
别想"嘲讽/光环/角色"这些大功能，想四个轴各选一个值来拼：

| 轴 | 在问 | 值菜单 |
|----|------|--------|
| **来源 Source** | 谁产生 | 背包物品 / 职业被动 / 技能 |
| **效果 Effect** | 产生啥 | +攻防血速魔暴击 / 治疗·护盾·回蓝 / 嘲讽 / 闪避(新) / 伤害转移(新) / 触发型(击杀回血·受击反伤) |
| **范围 Scope** ⭐ | 给谁 | 自己 / 全队 / **相邻** / **同排(相对)** / **前排·后排(绝对)** / 特定职业 / 单个友军(血最少) |
| **触发 Trigger** | 何时 | 常驻 / 战斗事件(回合начало·受击·击杀·友军死) / 条件(前排活着时·自身>50%) |

**Scope 是"小队"新增的那根轴**——把现有"自己"换成"全队/相邻/同排"，立刻有协同。

### 角色是涌现的，不是新系统
"常规前排"=高防属性+前排站位；"闪避T"=闪避效果+嘲讽+前排。**角色 = 上面原子的不同组合**，不用单独写职业系统。

### 深度原则：Scope 越窄越深
全队光环(放哪都生效)=无摆位决策=浅；**相邻/同排/条件光环**(军旗得放前排、疾风得挨着输出)=逼你冒险摆位=深。**优先用窄 scope。**

### 实现分两档
- **第一档·开战属性光环**（最便宜，先做）：效果是纯属性 → `BackpackLoadout.build_party` 加**第二遍扫描**：所有英雄建好、站位已知后，扫带 `team`/`aura` 字段的物品，按 scope 把属性加给对应单位。**不碰战斗引擎。**
  ```gdscript
  "war_banner":  { "name":"军旗", "team": {"atk": 4} }                       # 全队+4攻
  "speed_totem": { "name":"疾风图腾", "aura": {"scope":"adjacent","spd":3} }  # 相邻+3速
  ```
- **第二档·战斗行为型**（碰 BattleSimulator，靠后）：嘲讽(已部分有 `HAS_TAUNT`，做成物品可带) / **闪避**(新关键词，伤害结算前 roll 免伤) / 事件触发(`on_battle_event` 钩子已有)。

### 路线
- ✅ **第一切片（已做）站位光环**：物品 `aura: {scope, 属性...}`；`BackpackModel.grid_auras` 提取，`BackpackLoadout.build_party` 阶段2按 scope 注入。scope 支持 **team / adjacent / same_row(相对同排) / front_row·back_row(绝对前后排)**。已加 军旗(全队+攻)/疾风图腾(相邻+速)/铁壁旗(同排+防)/先锋号角(前排+攻)/守护图腾(后排+防血)；tooltip/商店显示光环；`PowerScore` 给光环算分。配 GUT(全队含自己/相邻/同排/前排/后排)。
  - **same_row vs front_row 区别**：same_row=跟持有者同排(相对)；front_row/back_row=指定前/后排(绝对，后排持有者也能 buff 全体前排)。
- **Backlog（第二档战斗钩子）**：① 闪避关键词(几率免伤，做"闪避T") ② 嘲讽做成物品/光环可带 ③ 伤害转移/分摊 ④ 事件触发型队级效果。
- **以后加光环料 = 给物品加 `aura` 字段一行**（scope 已支持 team/adjacent/same_row；要新 scope 在 `_aura_hits` 加分支）。
