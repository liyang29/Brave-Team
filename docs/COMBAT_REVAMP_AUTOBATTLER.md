## 0. 为什么要做这次重做（问题诊断）

白盒试玩结论：**当前 Demo 不好玩、无聊。** 根因不是美术缺失，而是乐趣链条断裂：

| 断点 | 现状 |
|------|------|
| ① 决策不够有意义 | 同职业英雄是可替换数值块，派遣 = "派当前属性最高的空闲队"，无取舍 |
| ② 结果没有悬念 | 难度星级 + 双方属性都可见，玩家心算即知胜负，"开箱"前已知答案 |
| ③ 后果不被在乎 | 永久死亡的是无名数值块，死了再招一个等价的 |
| ④ "看战斗"本身最无聊 | 一屏自动滚动的文字日志，玩家零操作，等于读表格自己算完 |
| ⑤ 节奏空转 | 实时制等队伍走过地图再走回来，中途无事可做 |

**最致命的是 ①②④ 叠加：玩家既无法影响结果、结果也无悬念、过程还是被动读日志。**

**本方案（B）的修复策略：** 不动"战斗不可干预"这条核心支柱，而是把全部深度压到**战斗前的"编队解谜"**上，并把战斗结算改成**快速、可归因的揭晓**——对标自走棋品类（云顶之弈 / Super Auto Pets / Mechabellum）已验证的公式：

> **被动看战斗不是病；"浅备战 + 慢结算"才是病。**
> 深备战（站位 + 克制 + 配装）→ 快揭晓（2~3 秒 + 结果卡归因）→ 紧循环。

## 1. 形态定位

**一句话：** 每个任务是一道**设计好的遭遇谜题**；玩家用「选人 + 前后排站位 + 配装/带药」求解，战斗自动结算并快速揭晓胜负与原因；英雄永久死亡让赌注真实。

**明确不做：**
- ❌ 随机商店 / 经济 roll / 天梯 PvP（那是另一种游戏，会丢掉公会 + 养成 + 永久死亡身份）
- ❌ 玩家在战斗中实时下指令（保留"撒手旁观"支柱；玩家指挥的是**布阵**，不是**操作**）
- ❌ 第一版就上自由网格（成本高、风险大；留作 B5 升级，地基预留）

**核心形态：小网格站位 + 回合制（Into the Breach 式，非即时）。** 玩家在战前把英雄摆到一个小网格上，行维度决定"前后排/触及"，列维度带来侧翼、AOE 形状与相邻连携；战斗按回合自动结算。
> 注：2 排（前/后排）是网格"行维度"的最小情形，已在脏实验中验证乐趣（见 §8），是通往网格的踏脚石。

**乐趣假设（脏实验已初步验证）：**
> 当玩家意识到"这一关得专门为它配队/布阵"，然后看到自己的安排奏效险胜的那一下——产生"我真聪明"的火花。

## 1.5 已确认决策（2026-06-14，方案评审）

经讨论与逐条确认，以下决策**已锁定**，后续章节若仍写"前/后排"，一律理解为"网格行维度的最小情形"，可推广到网格：

| 决策点 | 结论 | 理由 |
|--------|------|------|
| **时间模型** | **回合制（锁定）**；否决即时制自走棋 | 复用现有 `BattleSimulator`、谜面可读、可写测试、好平衡；即时制要推倒重写且会让谜题变模糊。**想要"实时观感"靠后期动画播放回合结果实现，不做真即时。** |
| **站位粒度** | **小网格 + 回合制** | 比 2 排深度更高（侧翼/AOE 形状/相邻连携）。代价是工作量更大 → 分小步增量推进，先在隔离实验场景做。 |
| **信息可读度** | **威胁标签级** | 战前显示危险"类型"（剧毒/重甲/后排法师/群攻），不给精确数值。解谜的前提；侦察可作为后期进阶。 |
| **第一波谜题轴** | **抗性 + 敌人威胁机制 + 装备即钥匙 + 解锁门控**（全做） | 低成本高收益、复用现有系统、能组合叠加。详见 §2.5。 |
| **红线** | 每根轴的解必须①战前看得见 ②玩家拿得到或解锁得到 | 否则是"恶心"而非"谜题"。 |

## 2. 三大支柱

整套玩法靠三根支柱撑起，缺一不可。**实现时不要只做支柱一。**

### 支柱一：站位（小网格 + 回合制）

- 每方一个**小网格**（建议起步 2 行 × 3 列，待定，见 §10）；玩家在编队界面把英雄摆进格子。
- **行维度（前/后）= 触及**：
  - **近战**（战士 / 盗贼 / 多数野怪）：只能攻击**敌方前排（前行）**，前排清空后才能打后排。
  - **远程**（弓手 / 法师 / 术士敌人）：可攻击任意行，按各自策略选目标。
- **列维度（左/中/右）= 战术形状**：
  - **侧翼**：某些技能/单位能绕列打到斜后方，或攻击同列纵深。
  - **AOE 形状**：横扫打一行、穿刺打一列、爆炸打 3×3——摆位决定能否一次多杀，也决定你的脆皮会不会被一锅端。
  - **相邻连携**：相邻格子的英雄可触发增益/组合（留作内容扩展）。
- **关键张力**：敌方会把**治疗 / 术士**放后排、把肉盾铺前排。玩家若没有"能够到后排/能拉成对的 AOE 阵型"的手段，就会被后排白嫖、或被敌方 AOE 一锅端。
  → "带谁、把谁摆哪一格"成为真正的决策。
- **现状**：脏实验已实现 2 排（行维度）触及规则并验证乐趣；**列维度 + 网格是接下来要做的核心升级**（B2）。

### 支柱二：克制网（角色 + 反制）

小而可读的"石头剪刀布"，**知道答案但执行不平凡**。

**敌方角色（role，新增 EnemyData 字段）：**

| role | 威胁 | 解法（克制） |
|------|------|------|
| `armored` 重甲 | 物理伤害大幅减免 | 法师魔法伤害 / 破甲技能 |
| `poison` 毒虫 | 持续掉血（DoT） | 牧师净化 / 高 HP 速杀 |
| `swarm` 群兽 | 数量压制、多段攻击 | 法师 AOE / 群体技能 |
| `caster` 术士（后排） | 后排高魔法爆发 | 盗贼突袭 / 弓手点名后排 |
| `bruiser` 狂战（前排） | 前排高攻 | 战士嘲讽吸收 + 减速 |
| `corrupted` 灵蚀造物 | 侵蚀：降低英雄属性/精神（接 PERSONALITY_SYSTEM） | 守望者祝福 / 高精神英雄 |

**英雄职业在本系统的定位：**

| 职业 | 排位 | 角色 | 解的题 |
|------|------|------|------|
| 战士 | 前排 | 嘲讽坦克 | bruiser、保护后排 |
| 牧师 | 后排 | 治疗 + **净化** | poison、续航 |
| 法师 | 后排 | AOE + 魔法伤害 | armored、swarm |
| 盗贼 | 前/后 | 高速 + **突袭后排** | caster、补刀 |
| 弓手 | 后排 | 稳定远程、点名 | caster、高攻目标 |

> 新克制手段（牧师净化 / 法师破甲 / 盗贼突袭）作为**技能**落到 `SkillTable`，不硬编码进 Simulator。

### 支柱三：可读性 + 快揭晓

解谜的前提是**信息可读**，乐趣的前提是**揭晓爽快**。

- **战前可读（已确认：威胁标签级）**：任务板 / 编队界面显示该遭遇的**威胁标签**（如「⚠️ 剧毒巢穴——无净化/低 HP 队伍易团灭」）和敌方角色构成（**显示危险类型，不给精确数值**；侦察可作为后期进阶，见 §1.5）。
- **快揭晓**：战斗**自动快进播放 2~3 秒**（血条 + 飘字），不再逐行读日志。
- **结果卡（归因）**：战斗结束弹一张卡，点出**为什么赢 / 为什么输**：
  - ✅「牧师净化了剧毒 → 全员扛住 → 险胜」
  - ❌「无人能触及敌方术士 → 后排被连续点名 → 团灭」
- 详细回合日志保留为**「展开查看」**的可选项（复用现有 `BattleUI`）。

## 2.5 谜题轴清单（深度来源）

**原则：深度来自多个简单"轴"叠加，不是单个复杂系统。** 单独一根轴玩两次就腻；3~4 根轴相乘（如"剧毒 + 重甲 + 后排召唤师"逼出"净化 + 破甲 + 够后排"的队）才耐玩。每根轴 = 造一种谜题、逼一种解。

**🔥 第一波（已确认，按低成本高收益排）：**

| 轴 | 造的谜题 | 需要的解 | 成本 |
|----|---------|---------|------|
| **物理/魔法抗性** | 重甲物免高 / 法抗怪魔免高 | 法师魔伤 / 物理 / 破甲穿透 | 低（`_calc_damage` 已分物理/魔法，`ignore_def` 已有）|
| **敌人威胁机制** | 敌方治疗者 / 狂暴(越打越强) / 护盾 | 爆发过量回血、速杀、破盾 | 低（`heal_ally` / 按回合 buff / `apply_shield` 多已有）|
| **装备即钥匙** | 用配装而非换人来解题 | 抗毒护符(自带净化)、破甲武器、嘲讽盾 | 低（装备触发系统现成，只造道具）|
| **解锁门控** | 现在打不过的谜题，解锁钥匙后能解 | 把"解"藏进公会升级/设施/招募 | 极低（纯内容编排，串主线，payoff 最高）|

**🧊 后续储备（评估后再排期）：**

| 轴 | 谜题 / 解 |
|----|----------|
| 元素抗性（火/冰/毒/光）| 带对应克制元素 |
| 召唤师 / 群兽 | AOE 清场 or 速杀召唤源 |
| 眩晕/沉默、反伤/荆棘 | 冗余输出 / 抢先手 / 大单发 |
| 站位深化（侧翼·护卫·相邻连携）| 网格的列维度玩法（见支柱一）|
| 胜利条件变化（撑 N 回合 / 限时斩杀 / 护送 / 按序击杀）| 跳出"打光就赢"，需改 Simulator 结束判定 |
| 玩家约束（限人数 / 禁某职业 / 地形修正）| 逼你跳出舒适配队 |
| 英雄连携（标记+引爆、增益+收割）| 凑组合技 |
| 战前战术指令（集火/保人/谨慎）| 一点临场判断（C 味道的安全注入）|

> **红线（每根轴都适用）**：① 它的解，玩家**战前看得见**（接威胁标签）；② 它的解，玩家**拿得到或解锁得到**（没法师却出"必须魔法关"= 耍流氓）。

## 4. 数据结构改动

### 4.1 `EnemyData`（`scripts/entities/EnemyData.gd`）

## 5. `BattleSimulator` 改动（核心）

改动**集中且局部**，主要在"取对手列表"和"选目标"两处：

### 5.1 触及过滤 `_get_reachable_opponents()`

`_get_opponents()` 之后增加一层排位过滤：

```gdscript
static func _get_reachable_opponents(unit, opponents) -> Array:
    # 远程/可达后排：全体可选
    if unit.can_reach_back:
        return opponents
    # 近战：敌方前排存活则只能打前排；前排清空才能打后排
    var front = opponents.filter(func(bc): return bc.row == "front")
    return front if not front.is_empty() else opponents
```

在 `simulate()` 选目标前调用：`var reachable = _get_reachable_opponents(unit, opponents)`，把 `reachable` 传给 `choose_target` 和 AOE 循环。
> 嘲讽（已有 `HAS_TAUNT` / `_find_taunt_target`）在 `reachable` 范围内优先生效，逻辑不冲突。

### 5.2 新克制机制（落 SkillTable，不硬编码）

- **净化**（牧师）：新技能 type `cleanse`——移除友军身上的 DoT（`active_effects` 里 dot 类），对 `poison` 遭遇是直接解。
- **破甲**（法师/技能）：`ignore_def` / `half_def` 已支持，给法师一个稳定破甲技即可克 `armored`。
- **突袭**（盗贼）：标记技能使该单位 `can_reach_back = true`（或开局即 true），直取敌方后排 `caster`。
- **侵蚀**（corrupted 敌人）：新 DoT/Debuff 变体，降低英雄 attack/speed（接精神系统时升级为降 mental_state）。

### 5.3 结果归因（给结果卡用）

`BattleResult` 增加 `key_events: Array`（如 `["cleanse_saved", "backline_unreachable"]`），由 Simulator 在关键节点 append，供结果卡生成归因文案。

## 8. 第一个脏实验（✅ 已完成 2026-06-14）

> **结果：机制层假设成立。** 同一批英雄换配队即结果反转——纯两战士 0/20 胜（够不到后排术士），+法师 19/20、+弓手 20/20（点掉术士），+牧师净化 15/20。可玩场景 `scenes/experiments/PositionExperiment.tscn`，回归测试 `tests/test_position_experiment.gd`，全量 258/258 绿。**待用户实玩给出"有没有火花"的主观判断后，按 §7 推进 B2（网格）。**

**目标：用最小代价让自己感受"配队解谜"的火花，不做完整 B0~B4。**

做法（约半天，不碰主流程，可随时回退）：

1. 手搓一个遭遇：敌方**后排 1 个「剧毒术士」(`caster`+`poison`) + 前排 2 个肉盾**。
2. 最小实现 B1 触及规则 + B2 的两点：牧师净化、盗贼/弓手能打后排。
3. 给这关一个威胁标签「⚠️ 剧毒术士藏在后排」。
4. **连玩两遍**：
   - 第一遍故意只带前排近战 → 够不到术士 + 被毒 → 团灭。
   - 第二遍带弓手/盗贼点掉术士 + 牧师净化 → 险胜。
5. 自问：**第二遍赢的那一下，有没有"我针对它配队 → 成功了"的火花？**

- 有火花 → B 方向成立，按 B0→B4 正式铺。
- 没火花 → 问题更深（也许"你不下场战斗"这个前提本身不适合），再回头讨论方案 C 或松动核心支柱。

## 9. 与世界观 / 既有系统的协同

- **守望者祝福**（GDD §3.11 / PERSONALITY_SYSTEM）= 战斗中**唯一**的有限干预杠杆：净化侵蚀 / 救一次人。既给玩家一丝 agency，又不破坏"撒手旁观"支柱——天然契合本方案。
- **灵蚀造物 `corrupted` 敌人**：把"侵蚀"威胁接到未来的 `mental_state`，让人格系统与战斗解谜互相喂养（看着"我的怪人"扛侵蚀 = 情感投射 + 解谜双重乐趣）。
- **永久死亡**：站位失误的代价是失去有名有姓的英雄 → 把"备战解谜"的赌注情绪拉满。
